1 nanocoap server example
2 ========================
3 
4 This application is meant to get you started with implementing a CoAP server on RIOT.
5 It uses the GNRC network stack through RIOT's
6 [sock API](http://doc.riot-os.org/group__net__sock.html).
7 
8 Usage
9 =====
10 
11 To try out the server on native, compile it with
12 
13 ```
14 $ make all
15 ```
16 
17 Then, create a tap interface (to which RIOT will connect):
18 
19 ```
20 $ sudo ip tuntap add tap0 mode tap user ${USER}
21 $ sudo ip link set tap0 up
22 ```
23 
24 Run the resulting RIOT binary by invoking:
25 
26 ```
27 $ make term
28 ```
29 
30 The application is now listening on all it's configured IP addresses.

63 Testing
64 =======
65 There are multiple external CoAP clients you can use to test the server on native.
66 
67 libcoap CLI
68 -----------
69 
70 (replace "fe80::e42a:1aff:feca:10ec" with your link-layer address)
71 
72 ```
73 # coap-client "coap://[fe80::e42a:1aff:feca:10ec%tap0]/riot/board"
74 ```

76 Copper (Firefox Plugin)
77 -----------------------
78 
79 The Copper plugin for Firefox provides you with a nice graphical interface, but
80 getting it to work with RIOT requires a little setup.
81 
82 Make sure you've installed
83 
84 - The [Firefox Copper plugin](https://addons.mozilla.org/en-US/firefox/addon/copper-270430/)
85 - The Router Advertisement Daemon (radvd)
86 
87 And build the application again using `make`.
88 
89 Enter the following into your `/etc/radvd.conf` (if it doesn't exist yet, create one):
90 
91 ```
92 interface tap0
93 {
94  AdvSendAdvert on;
95 
96  MinRtrAdvInterval 3;
97  MaxRtrAdvInterval 10;
98 
99  AdvDefaultPreference low;
100 
101  prefix 2001:db8:1:0::/64
102  {
103  AdvOnLink on;
104  AdvAutonomous on;
105  AdvRouterAddr off;
106  };
107 };
108 ```
109 
110 (you can use `radvd -c` to check for syntax errors)
111 
112 and run
113 
114 ```
115 sudo radvd
116 ```
117 
118 Then, run the RIOT binary as usual:
119 
120 ```
121 make term
122 ```
123 
124 Note that the output listing all configured interfaces contains a globally scoped
125 address, which you can now use to reach the RIOT instance via Copper. To do so, enter this:
126 
127 ```
128 coap://[2001:db8:1:0:e42a:1aff:feca:10ec]/riot/board
129 ```
130 
131 into your Firefox address bar, where you should replace `2001:db8:1:0:e42a:1aff:feca:10ec`
132 with your RIOT instance's address marked as "scope: **global**".
133 If you click the big green `GET` button, the word `native` should appear in the
134 **Payload** text box at the center of the GUI.
135 
136 **If this doesn't work,** try manually adding a Global address to the tap0 interface:
137 
138 ```
139 sudo service radvd start
140 sudo ip address add 2001:db8:1::a/64 dev tap0
141 make term
142 ```
