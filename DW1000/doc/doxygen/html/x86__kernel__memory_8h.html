◆ _heap_start

char _heap_start

First byte that may be mapped as the heap.

This symbol is expected to be aligned. Must be bigger than _kernel_memory_end. There should be a space between _kernel_memory_end and _heap_start, but it does not have to.

◆ _kernel_memory_end

char _kernel_memory_end

First byte after the last of the common elf segments.

This symbol is expected to be aligned. The "common elf segments" are .text, .rodata, .data and .bss. All the (physical) memory beyond this point will be overwritten at the pleasure of the kernel.

◆ _kernel_memory_start

char _kernel_memory_start

First byte inside the elf segments.

Expected to be the start of the high memory, i.e. 1MB.

◆ _section_bss_end

char _section_bss_end

First byte after the .bss segment.

This symbol should not to be aligned, because RAM cycles would be wasted to clear unused bytes otherwise.

◆ _section_bss_start

char _section_bss_start

Start of the .bss segment.

Probably unaligned. The .bss segment must follow the .data segment with no other segment inbetween. It's likely that &_section_data_end == &_section_bss_start.
