1 examples/default
2 ================
3 This application is a showcase for RIOT's hardware support. Using it
4 for your board, you should be able to interactively use any hardware
5 that is supported.
6 
7 To do this, the application uses the `shell` and `shell_commands`
8 modules and all the driver modules each board supports.
9 
10 `shell` is a very simple interactive command interpreter that can be
11 used to call functions. Many of RIOT's modules define some generic
12 shell commands. These are included via the `shell_commands` module.
13 
14 Additionally, the `ps` module which provides the `ps` shell command is
15 included.
16 
17 Usage
18 =====
19 
20 Build, flash and start the application:
21 ```
22 export BOARD=your_board
23 make
24 make flash
25 make term
26 ```
27 
28 The `term` make target starts a terminal emulator for your board. It
29 connects to a default port so you can interact with the shell, usually
30 that is `/dev/ttyUSB0`. If your port is named differently, the
31 `PORT=/dev/yourport` variable can be used to override this.
