3 ## Connecting RIOT native and the Linux host
4 
5 > **Note:** RIOT does not support IPv4, so you need to stick to IPv6 anytime. To establish a connection between RIOT and the Linux host,
6 you will need `tftp` (with IPv6 support). On Ubuntu and Debian you would need the package `tftp-hpa` for TFTP client and `tftpd-hpa` for TFTP server.
7 Be aware that many programs require you to add an option such as -6 to tell them to use IPv6, otherwise they
8 will fail. If you're using a _Raspberry Pi_, run `sudo modprobe ipv6` before trying this example, because raspbian does not load the
9 IPv6 module automatically.
10 On some systems (openSUSE for example), the _firewall_ may interfere, and prevent some packets to arrive at the application (they will
11 however show up in Wireshark, which can be confusing). So be sure to adjust your firewall rules, or turn it off (who needs security anyway).
12 
13 First, create a tap interface (to which RIOT will connect) and a bridge (to which Linux will connect) from the RIOT main directory run:
14 
15  ./dist/tools/tapsetup/tapsetup -c 1
16 
17 Now you can start the `gnrc_tftp` example by invoking `make term`. This should automatically connect to the `tap0` interface. If
18 this doesn't work for some reason, run `make` without any arguments, and then run the binary manually like so (assuming you are in the `examples/gnrc_tftp` directory):
19 
20 To verify that there is connectivity between RIOT and Linux, go to the RIOT console and run `ifconfig`:
21 
22  > ifconfig
23  Iface 6 HWaddr: 7e:ed:d2:ee:e1:07
24  MTU:1280
25  Source address length: 6
26  Link type: wired
27  inet6 addr: ff02::1/128 scope: local [multicast]
28  inet6 addr: fe80::7ced:d2ff:feee:e107/64 scope: local
29  inet6 addr: ff02::1:ffee:e107/128 scope: local [multicast]
30 
31 
32 Copy the [link-local address](https://en.wikipedia.org/wiki/Link-local_address) of the RIOT node (prefixed with `fe80`) and try to ping it **from the Linux node**:
33 
34  ping6 fe80::7ced:d2ff:feee:e107%tapbr0
35 
36 Note that the interface on which to send the ping needs to be appended to the IPv6 address, `%tapbr0` in the above example. When talking to the RIOT node, you always want to send to/receive from the `tapbr0` interface.
37 
38 If the pings succeed you can go on to send UDP packets. To do that, first start a UDP server on the RIOT node:
39 
40  > tftps start
41  tftp_server: Starting TFTP service at port 69
42 
43 Now, on the Linux host, you can run tftp to connect with RIOT's TFTP server:
44 
45  $ tftp -v -6 fe80::7ced:d2ff:feee:e107%tapbr0 -c get welcome.txt
46 
47 The output will be something like:
48 
49  Connected to fe80::7ced:d2ff:feee:e107%tapbr0 (fe80::7ced:d2ff:feee:e107), port 69
50  getting from fe80::7ced:d2ff:feee:e107%tapbr0:welcome.txt to welcome.txt [netascii]
51  Received 94 bytes in 0.0 seconds [113425 bit/s]
52 
53 The `-6` option is necessary to tell tftp to use IPv6 only and the `-v` is to tell the tftp client to be verbose.
54 
55 You should now see that the TFTP messages are received on the RIOT side. Opening a TFTP server on the Linux side is also possible. To do that, write down the IP address of the host (run on Linux):
56 
57  ifconfig tapbr0
58  tapbr0 Link encap:Ethernet HWaddr 0e:bc:0f:49:7f:e4
59  inet6 addr: fe80::cbc:fff:fe49:7fe4/64 Scope:Link
60  UP BROADCAST RUNNING MULTICAST MTU:1500 Metric:1
61  RX packets:22 errors:0 dropped:0 overruns:0 frame:0
62  TX packets:53 errors:0 dropped:0 overruns:0 carrier:0
63  collisions:0 txqueuelen:0
64  RX bytes:1928 (1.9 KB) TX bytes:6217 (6.2 KB)
65 
66 And start the tftp server on Linux:
67 
68  $ sudo /etc/init.d/tftpd-hpa stop && mkdir -p tftpserv && echo "hello world" > tftpserv/welcome.txt && sudo in.tftpd -vvv -L -6 -c -s -u ${USER} ./tftpserv
69 
70 Now, on the RIOT side, send a UDP packet using:
71 
72  tftpc get welcome.txt octet 1 fe80::cbc:fff:fe49:7fe4
73
