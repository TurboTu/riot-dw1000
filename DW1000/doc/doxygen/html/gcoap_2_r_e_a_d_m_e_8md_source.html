2 
3 This application provides command line access to gcoap, a high-level API for CoAP messaging. See the [CoAP spec][1] for background, and the Modules>Networking>CoAP topic in the source documentation for detailed usage instructions and implementation notes.

5 We support two setup options for this example:
6 
7 ### Native networking
8 Build with the standard `Makefile`. Follow the setup [instructions][2] for the gnrc_networking example.

10 ### SLIP-based border router
11 
12 Build with `Makefile.slip`. Follow the setup instructions in README-slip.md, which are based on the [SLIP instructions][3] for the gnrc_border_router example. We also plan to provide or reference the ethos/UHCP instructions, but we don't have it working yet.

14 ## Current Status
15 gcoap includes server and client capability. Available features include:
16 
17 * Server and Client provide helper functions for writing the response/request. See the CoAP topic in the source documentation for details. See the gcoap example for sample implementations.
18 * Server allows an application to register a 'listener', which includes an array of endpoint paths and function callbacks used to write a response.
19 * Server listens on a port at startup; defaults to 5683.
20 * Client operates asynchronously; sends request and then handles response in a user provided callback. Also executes callback on timeout.
21 * Client generates token; length defined at compile time.
22 * Message Type: Supports non-confirmable (NON) messaging.
23 * Options: Supports Content-Format for response payload.

26 ## Example Use
27 This example uses gcoap as a server on RIOT native. Then we send a request from a libcoap example client on the Linux host.
28 
29 ### Verify setup from RIOT terminal
30 
31  > coap info
32 
33 Expected response:

36  CLI requests sent: 0

39 ### Query from libcoap example client
40 gcoap does not provide any output to the CoAP terminal when it handles a request. We recommend use of Wireshark to see the request and response. You also can add some debug output in the endpoint function callback.
41 
42  ./coap-client -N -m get -p 5683 coap://[fe80::1843:8eff:fe40:4eaa%tap0]/.well-known/core
43 
44 Example response:
45 
46  v:1 t:NON c:GET i:0daa {} [ ]
47  </cli/stats>
48 
49 The response shows the endpoint registered by the gcoap CLI example.
