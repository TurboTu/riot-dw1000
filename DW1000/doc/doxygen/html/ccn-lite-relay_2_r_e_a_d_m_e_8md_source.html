1 # CCN-Lite on RIOT
2 
3 This application demonstrates how to use the Content-Centric Networking stack
4 from [CCN-Lite](http://www.ccn-lite.net/) on RIOT. In the current state you can
5 use only one packet format, `ndn2013`, and only relay over the link-layer
6 (Ethernet, IEEE~802.15.4 or what else is supported by your network device).
7 
8 ## The shell commands
9 
10 RIOT provides three shell to interact with the CCN-Lite stack:
11 * `ccnl_int` - generates and sends out an Interest. The command expects one
12  mandatory and one optional parameter. The first parameter
13  specifies the exact name (or a prefix) to request, the second
14  parameter specifies the link-layer address of the relay to use.
15  If the second parameter is omitted, the Interest will be
16  broadcasted. You may call it like this:
17  `ccnl_int /riot/peter/schmerzl b6:e5:94:26:ab:da`
18 * `ccnl_cont` - generates and populates content. The command expects one
19  mandatory and one optional parameter. The first parameter
20  specifies the name of the content to be created, the second
21  parameter specifies the content itself. The second parameter
22  may include spaces, e.g. you can call:
23  `ccnl_cont /riot/peter/schmerzl Hello World! Hello RIOT!`
24 * `ccnl_fib` - modifies the FIB or shows its current state. If the command is
25  called without parameters, it will print the current state of
26  the FIB. It can also be called with the action parameters `add`
27  or `del` to add or delete an entry from the FIB, e.g.
28  `ccnl_fib add /riot/peter/schmerzl ab:cd:ef:01:23:45:67:89`
29  will add an entry for `/riot/peter/schmerzl` with
30  `ab:cd:ef:01:23:45:67:89` as a next hop and
31  `ccnl_fib del /riot/peter/schmerzl`
32  will remove the entry for `/riot/peter/schmerzl` and
33  `ccnl_fib del ab:cd:ef:01:23:45:67:89`
34  will remove all entries with `ab:cd:ef:01:23:45:67:89` as a
35  next hop.

37 ## Example setup
38 
39 An example usage of this application could be setup like this:
40 1. Open a terminal window, navigate to the RIOT directory, and enter
41  `dist/tools/tapsetup/tapsetup -c`.
42 2. Open a second terminal window and navigate to this directory in both of
43  windows.
44 3. Call `make -B clean all term` in the first terminal and `PORT=tap1 make
45  term` in the second one.
46 4. Enter `ccnl_cont /riot/peter/schmerzl Hello World! Hello RIOT!` on the first
47  terminal.
48 5. Add a FIB entry for this prefix on the second node, e.g. using the broadcast
49  address: `ccnl_fib add /riot/peter/schmerzl ff:ff:ff:ff:ff:ff`
50 6. Enter `ccnl_int /riot/peter/schmerzl` in the second terminal.
51 7. See the content being displayed. Be happy!

53 ## Makefile configuration
54 
55 The ccn-lite package provides several configuration options through defines
56 that can be set in the application Makefile. The following options are
57 mandatory for now:
58 * `CFLAGS += -DUSE_LINKLAYER` - use CCN directly over the link layer
59 * `CFLAGS += -DCCNL_UAPI_H_` - tell ccn-lite to use the UAPI
60 * `CFLAGS += -DUSE_SUITE_NDNTLV` - use NDNTLV packet format
61 * `CFLAGS += -DNEEDS_PREFIX_MATCHING` - enables prefix matching
62 * `CFLAGS += -DNEEDS_PACKET_CRAFTING` - enable userspace packet creation
63 Here's a list of some additional interesting options:
64 * `CFLAGS += -DUSE_RONR` - enable Reactive Optimistic Name-based
65  Routing (RONR)
66 * `CFLAGS += -DUSE_STATS` - enable statistics
67 * `CFLAGS += -DUSE_DUP_CHECK` - enable duplicate checks when forwarding
68 * `CFLAGS += -DUSE_HMAC256` - HMAC256 signed packets for CCNx1.0
69  encoding

72 ## Wireshark dissector
73 
74 One can use the Wireshark dissector from the named-data project
75 (https://github.com/named-data/ndn-tools/tree/master/tools/dissect-wireshark)
76 for analyzing the traffic in Wireshark.
77 
78 However, please note, that - in order to be compatible with the default
79 CCN-Lite upstream configuration - a different Ethertype (`0x0801` instead of
80 `0x8624`) is used.
81 
82 The simplest way to get this working is to copy the `ndn.lua` file into your
83 local Wireshark plugin directory (e.g. `$HOME/.wireshark/plugins`) and update
84 https://github.com/named-data/ndn-tools/blob/master/tools/dissect-wireshark/ndn.lua#L424
85 to `0x0801).
